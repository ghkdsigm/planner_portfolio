
## 1. DW-Brain · AI 개입형 브레인스토밍 협업 채팅 데스크톱앱 & WEB

1.2. 문제 정의
업무 현장에서 회의, 브레인스토밍, 협업 채팅, 문서 작성, AI 활용 등 다양한 작업이 이루어지고 있으나,
각 과정이 서로 다른 도구 안에서 개별적으로 진행되면서 사고 흐름이 하나의 구조로 연결되지 않는 문제 존재함.

아이디어는 메신저나 채팅에 분산되고,
정리는 별도의 문서로 다시 작성되며,
AI는 필요 시 개별적으로 호출하는 보조 도구로 사용되는 구조가 일반적임.

이러한 환경에서는 브레인스토밍 과정 자체가 기록되지 않고,
의사결정의 흐름이 축적되지 않으며,
이미 논의된 내용이 반복되는 경우가 많고,
협업 과정이 실제 실행 가능한 결과로 이어지지 않는 비효율 발생함.

또한 협업 과정이 데이터로 남지 않는 구조에서는
어떤 논의가 어떤 결과로 이어졌는지,
어떤 아이디어가 성과에 영향을 주었는지,
어떤 의사결정이 실패의 원인이었는지에 대한 인과관계 추적이 어려움.

이로 인해 문제의 원인 분석이 어렵고,
개선 방향이 경험이나 개인 판단에 의존하게 되며,
동일한 시행착오가 반복되는 구조 발생함.

특히 협업툴, 문서툴, AI툴이 분리되어 있는 현재의 업무 환경에서는
대화 → 아이디어 → 정리 → 문서 → 실행으로 이어지는 흐름이 단절되기 쉬우며,
브레인스토밍 과정이 데이터로 축적되지 않아
AI를 활용하더라도 개인 단위 효율 향상에 그칠 가능성 존재함.

결과적으로 조직 단위의 의사결정 속도, 실행력, 지식 축적을 높이는 데 한계 존재함.

따라서 협업 과정 자체를 하나의 데이터 흐름으로 기록하고,
대화, 아이디어, 결정, 문서, 실행까지의 전 과정을 연결하며,
해당 과정에 AI가 실시간으로 개입하여 사고 흐름을 정리하고 구조화할 수 있는 새로운 협업 방식 필요함.

1.2. 해결전략설계
분산되어 있던 협업 과정과 브레인스토밍 흐름을 하나의 구조 안에서 연결하기 위해,
대화, 아이디어, 문서, 스펙, 실행 과정을 하나의 시스템 내에서 관리할 수 있는 협업 구조로 설계함.

채팅, 문서, AI 기능을 각각 사용하는 기존 방식에서 벗어나,
대화 자체가 기록되고 구조화되며 결과로 이어지는 흐름을 만들기 위해
채팅 기반 협업 환경에 AI가 실시간으로 개입하는 구조로 설계함.

브레인스토밍 과정에서 생성되는 대화, 아이디어, 결정 내용을 자동으로 정리하고,
문서, 스펙, 지식 형태로 전환될 수 있도록
대화 → 아이디어 → 문서 → 스펙 → 실행으로 이어지는 데이터 흐름 구조를 구성함.

또한 협업 과정 전체를 로그 형태로 축적하여
어떤 논의가 어떤 결과로 이어졌는지 추적 가능하도록 하고,
의사결정 과정과 결과 간의 인과관계를 분석할 수 있는 구조로 설계함.

AI는 단순 보조 도구가 아니라 협업 과정에 직접 개입하는 역할로 정의하고,
대화 정리, 아이디어 구조화, 문서 생성, 스펙 생성, 지식 정리 등을 자동으로 수행하도록 설계함.

이를 통해 개인 단위의 생산성 향상을 넘어
조직 단위의 사고 흐름을 기록하고,
지식을 축적하며,
의사결정 속도를 높일 수 있는 협업 환경을 구축하는 것을 목표로 설계함.

DW-Brain은 단순한 채팅 도구가 아니라,
조직의 사고 과정과 협업 흐름을 데이터로 축적하고 확장할 수 있는
AI 개입형 브레인 시스템을 지향하도록 구조를 정의함.

1.3. 해결
설계한 협업 구조의 실제 적용 가능성을 검증하기 위해
채팅 기반 협업 환경에 AI가 직접 개입하는 형태의 데스크톱 앱 및 WEB 프로토타입을 구현함.

대화, 아이디어, 문서, 스펙, 실행 흐름이 하나의 시스템 안에서 연결되는 구조를 확인하기 위해
채팅 인터페이스, AI 응답 구조, 문서 생성 기능, 지식 저장 구조를 포함한 기본 협업 기능을 구현함.

브레인스토밍 과정에서 생성되는 대화를 기반으로
AI가 내용을 정리하고,
문서 형태로 변환하며,
스펙 및 실행 단위로 확장할 수 있는 흐름을 실제 동작하도록 구성함.

또한 협업 과정의 로그를 저장하고,
대화 → 결정 → 결과로 이어지는 흐름을 추적할 수 있도록
데이터 축적 구조를 함께 구현함.

AI가 단순 응답 기능이 아니라
대화 흐름에 개입하여 정리, 요약, 구조화, 문서화, 스펙화까지 수행할 수 있는지 검증하고,
협업 생산성 향상에 실제로 기여할 수 있는지 테스트함.

프로토타입 구현과 내부 사용 테스트를 통해
기존 협업 방식 대비 사고 흐름 정리 속도,
문서 생성 시간,
아이디어 정리 효율이 개선되는 것을 확인함.

이를 통해 브레인스토밍 과정 자체를 데이터로 축적하고,
AI가 실시간으로 개입하여 협업 방식을 변화시킬 수 있는 가능성을 확인함.

DW-Brain은 단순 기능 구현이 아니라,
조직의 사고 흐름을 기록하고 실행으로 연결하는 협업 플랫폼으로 확장 가능함을 검증함.

## 2. AI 360 파노라마 인테리어 제너레이터

2.1 전략 정의

3D 및 VR 기반 인테리어 컨텐츠 제작 과정에서는
실제 공간을 모델링하거나 시뮬레이션 이미지를 제작하기 위해
촬영, 모델링, 텍스처 적용, 렌더링 등 많은 공정이 필요하며
제작 비용과 시간이 크게 소요되는 문제가 존재함.

특히 공간 기반 컨텐츠 제작은
외주 모델링, 렌더링 작업, 후처리 과정에 대한 비용 부담이 커
대량의 컨텐츠를 제작하기 어려운 구조임.

360 파노라마 촬영 기술과 AI 이미지 처리 기술을 결합하면 실제 공간을 기반으로 한 인테리어 컨텐츠를 3D 모델링 없이 생성할 수 있으며 VR / 360 컨텐츠 제작 비용을 크게 줄일 수 있다고 판단함.

또한 경쟁사 대비 AI 기반 이미지 자체 합성 기술과 자동 컨텐츠 생성 기술을 확보하면 초기 기술 진입장벽을 만들 수 있고
향후 XR / VR / 시뮬레이션 컨텐츠 확장 기반을 마련할 수 있다고 판단함.

따라서 360 촬영 데이터를 기반으로 공간 영역을 자동 인식하고 제품 텍스처를 자동 합성하여
VR / 360 / 이미지 컨텐츠를 생성할 수 있는 AI 기반 파노라마 인테리어 생성 시스템 구축을 목표로 프로젝트를 기획함.

2.2 문제 정의

기존 3D 및 VR 기반 인테리어 컨텐츠 제작 과정에서는
공간 모델링, 텍스처 적용, 렌더링 작업이 필요하며
제작에 많은 시간과 비용이 소요되는 문제가 존재함.

실제 공간을 기반으로 한 컨텐츠를 제작하기 위해서는
3D 모델링 작업 또는 이미지 편집 작업을 반복해야 하며
수정 요청 발생 시 동일 작업을 다시 수행해야 하는 비효율 발생함.

특히 다양한 제품을 동일 공간에 적용해야 하는 경우
텍스처 합성 작업을 반복 수행해야 하므로
컨텐츠 제작 속도가 크게 저하되는 문제가 있음.

360 파노라마 촬영을 활용하더라도
벽, 바닥, 천장 영역을 수동으로 마스킹해야 하고
제품 텍스처를 직접 합성해야 하며
결과 이미지 제작에 많은 작업 시간이 필요함.

또한 기존 컨텐츠 제작 방식에서는
촬영 데이터, 합성 결과, 텍스처 적용 정보가
하나의 구조로 관리되지 않아
자동화 및 재사용이 어려운 구조임.

이러한 환경에서는
대량의 VR / 360 / 인테리어 시뮬레이션 컨텐츠 제작이 어렵고
AI 기반 자동 생성 기술을 활용하기 어려운 한계 존재함.

따라서
360 촬영 데이터를 기반으로
공간 영역을 자동 인식하고
텍스처를 자동 합성하며
컨텐츠를 빠르게 생성할 수 있는
AI 기반 자동 인테리어 생성 구조 필요함.

2.3 해결 전략 설계

360 파노라마 촬영 데이터를 기반으로
공간 구조를 자동 인식하고 텍스처를 합성할 수 있도록
촬영 → 업로드 → 분석 → 합성 → 결과 생성까지
하나의 파이프라인 구조로 설계함.

Insta360 촬영 장비를 활용하여
실제 공간을 파노라마 이미지로 촬영하고
촬영 데이터를 서버로 자동 전송하는 구조로 구성함.

이미지 업로드 후
벽, 바닥, 천장 영역을 자동으로 구분할 수 있도록
AI 기반 영역 인식 및 마스킹 구조를 설계함.

제품 텍스처 데이터를 라이브러리 형태로 관리하고
선택한 제품을 특정 영역에 자동 적용할 수 있도록
텍스처 합성 엔진을 설계함.

동일 공간에 대해 여러 제품을 빠르게 적용할 수 있도록
이미지 생성 자동화 구조를 구성하고
결과 이미지를 반복 생성할 수 있도록 설계함.

생성된 결과 이미지는
360 컨텐츠, VR 컨텐츠, 이미지 시뮬레이션 등에 활용할 수 있도록
데이터 저장 및 재사용 구조를 함께 설계함.

본 구조를 통해
3D 모델링 중심 제작 방식에서
AI 기반 이미지 합성 중심 제작 방식으로 전환하는 것을 목표로 설계함.

2.4 실행

설계한 구조의 실제 적용 가능성을 검증하기 위해
360 파노라마 촬영, 이미지 업로드, 영역 인식, 텍스처 합성 기능을 포함한
AI 인테리어 생성 프로토타입을 구현함.

Insta360 촬영 데이터를 서버로 전송하고
이미지 영역을 분리한 후
제품 텍스처를 자동 적용하여
결과 이미지를 생성하는 흐름을 실제로 동작하도록 구성함.

벽 / 바닥 영역을 자동 구분하고
동화자연마루 및 벽지 텍스처를 합성할 수 있도록 구현하고
여러 제품을 반복 적용할 수 있는 테스트 환경을 구성함.

프로토타입 테스트를 통해
기존 3D / 이미지 편집 방식 대비
컨텐츠 제작 시간 감소,
반복 작업 감소,
수정 대응 속도 향상을 확인함.

또한 실제 공간 기반 360 이미지를 활용하여
VR / 360 컨텐츠 제작에 활용 가능함을 검증함.

AI 기반 이미지 합성 구조를 통해
향후 XR / VR / 시뮬레이션 컨텐츠로 확장 가능한 기술 기반을 확보함.

본 프로젝트를 통해
AI 기반 자동 컨텐츠 생성 기술을 확보하고
경쟁사 대비 초기 기술 진입장벽을 만들 수 있는 가능성을 확인함.

## 3. AI 스마트 카메이트 RAG 상담 시스템
3.1 전략 정의

중고차 거래 시장에서는
차량 정보 조회, 상담, 가격 확인, 매물 검색 등의 업무가 반복적으로 발생하며
기존 시스템은 단순 조회 중심 구조로 되어 있어
상담 자동화 및 지능형 검색 기능이 부족한 한계가 존재함.

기존에는 B2C 상담 AI 서비스로 기획했으나
LLM 기반 서비스의 토큰 사용량 증가로 인해
대량 사용자 대상 운영 시 비용 부담이 크게 발생하는 문제가 있었음.

이에 따라 타겟을 일반 소비자가 아닌
내부 고객인 단지 입점 딜러 및 상사를 대상으로 전환하고
업무 효율을 높이는 B2B AI 상담 시스템으로 방향을 재설정함.

특히 단지 내 약 10만대 이상의 차량 재고 데이터를 활용할 수 있는 환경을 기반으로
데이터 중심 AI 상담 서비스를 구축하면
기존 독점 시장인 카매니저 시스템과 차별화된 경쟁력을 확보할 수 있다고 판단함.

AI 기반 RAG 검색 및 상담 구조를 통해
차량 정보 조회, 매물 검색, 상담 응답을 자동화하고
구독형 서비스 모델로 확장할 수 있는 플랫폼 구축을 목표로 프로젝트를 기획함.

또한 향후 해외 중고차 수출 시장 대응을 위해
다국어 상담 기능, 수출업자 대상 기능, 차량 평가 및 밸류에이션 시스템까지 확장 가능한 구조 확보를 목표로 설정함.

3.2 문제 정의

기존 중고차 거래 시스템은
차량 정보 조회 및 관리 중심으로 구성되어 있어
상담, 검색, 추천, 비교 등의 업무를 자동화하기 어려운 구조임.

딜러 및 상사 업무에서는
차량 재고 조회, 옵션 확인, 가격 비교, 고객 상담을 반복 수행해야 하며
정보 검색과 응답에 많은 시간이 소요되는 문제가 있음.

또한 기존 시스템은
자연어 기반 검색이나 AI 상담 기능이 부족하여
사용자가 원하는 정보를 빠르게 찾기 어려운 한계 존재함.

특히 카매니저와 같은 기존 시스템이 시장을 독점하고 있어
단순 기능 개선만으로는 경쟁력을 확보하기 어려운 구조임.

B2C 대상 AI 상담 서비스로 운영할 경우
LLM 토큰 사용량 증가로 인해 운영 비용이 급격히 증가하며
서비스 지속 가능성이 낮아지는 문제가 발생함.

또한 차량 데이터가 대량으로 존재함에도
해당 데이터를 활용한 지능형 검색 및 상담 시스템이 부족하여
데이터 경쟁력을 충분히 활용하지 못하는 상황임.

향후 해외 수출, 다국어 상담, 차량 평가, 가격 산정 등
고도화 기능을 고려할 때
데이터 기반 AI 플랫폼 구조가 필요하다고 판단함.

따라서
대량 차량 데이터를 기반으로
RAG 검색, AI 상담, 자동 응답 기능을 제공할 수 있는
B2B 대상 AI 상담 플랫폼 필요함.

3.3 해결 전략 설계

단지 내 차량 재고 데이터를 기반으로
자연어 검색과 AI 상담이 가능하도록
RAG 기반 AI 상담 구조로 시스템을 설계함.

차량 정보, 옵션, 가격, 재고 데이터를
검색 가능한 형태로 구성하고
LLM과 연동하여 질문 기반 응답이 가능하도록 설계함.

토큰 사용량을 고려하여
불특정 다수 대상 서비스가 아닌
단지 입점 딜러 및 상사를 대상으로 하는
B2B 구독형 서비스 구조로 방향을 설정함.

기존 카매니저 중심 시장에서 경쟁하기 위해
단순 조회 기능이 아닌
AI 상담, 추천, 검색 자동화 기능을 중심으로 설계함.

RAG 구조를 적용하여
대량 차량 데이터를 기반으로 정확한 응답이 가능하도록 구성하고
상담 정확도를 높일 수 있는 데이터 파이프라인을 설계함.

또한 향후 확장을 고려하여 다국어 상담 기능 해외 수출업자 대상 기능 차량 평가 및 밸류에이션 시스템
가격 분석 기능을 추가할 수 있도록 확장 가능한 구조로 설계함.

이를 통해 단순 차량 관리 시스템이 아닌 데이터 기반 AI 중고차 플랫폼으로 발전할 수 있도록 설계함.

3.4 실행

RAG 기반 AI 상담 구조의 실제 적용 가능성을 검증하기 위해 차량 재고 데이터를 기반으로
검색, 질문, 응답 기능을 포함한 AI 상담 프로토타입을 구현함.

차량 정보 데이터를 벡터화하여 저장하고 사용자의 질문에 대해 관련 차량 데이터를 검색 후 LLM 응답을 생성하는 구조로 구성함.

단지 입점 상사 및 딜러를 대상으로 소수 사용자 테스트를 진행하고 실제 업무 환경에서 활용 가능성을 검증함.
구독형 서비스 모델을 적용하여 일부 상사 및 딜러가 실제로 사용하도록 테스트 환경을 구성하고 상담 속도, 검색 편의성, 정보 조회 효율 개선 여부를 확인함.
테스트 결과 기존 시스템 대비 검색 속도 향상, 상담 응답 시간 감소, 정보 조회 편의성 개선에 대한 긍정적인 평가를 확인함.

또한 10만대 이상 차량 재고 데이터를 기반으로 데이터 중심 AI 상담 서비스의 경쟁력을 확보할 수 있음을 확인함.

향후 해외 중고차 수출 대응을 위해 다국어 상담 기능, 수출업자 대상 기능, 차량 평가 및 밸류에이션 시스템 개발을 고려하고 있으며

본 프로젝트를 통해 기존 독점 시장 구조에 기술 기반으로 진입할 수 있는 가능성을 확인하고 데이터 기반 AI 플랫폼으로 확장 가능한 기반을 확보함.

## 4. Dongwha AI Orchestration Platform (DAOP)
4.1 전략 정의

전사 업무 환경에서는
다양한 시스템, 데이터, 문서, 프로세스가 분산되어 운영되고 있으며
업무 자동화 및 AI 활용이 개별 단위로 이루어져
전사 차원의 효율 개선으로 이어지기 어려운 구조가 존재함.

기존 자동화 방식은
개발 조직 중심으로 구현되거나
특정 시스템에 종속되는 경우가 많아
현업 사용자가 직접 자동화를 구성하기 어려운 한계가 있음.

AI 기능 또한
단일 서비스 단위로 제공되는 경우가 많아
업무 흐름 전체를 자동화하거나
여러 시스템을 연결하는 오케스트레이션 구조가 부족함.

전사 구성원이 직접
데이터, 정책, 액션을 조합하여
업무 프로세스를 자동화하고
AI 에이전트를 활용할 수 있는 플랫폼이 필요하다고 판단함.

따라서
데이터 리소스, 정책, 액션을 조립하여
에이전트를 생성하고
프로그램을 자동 생성하며
업무 자동화와 협업을 동시에 수행할 수 있는
AI Orchestration Platform 구축을 목표로 프로젝트를 기획함.

4.2 문제 정의

전사 시스템 환경에서는 업무 자동화가 개별 시스템 단위로 이루어지고 있어 데이터 사일로 환경에서 
프로세스 전체를 연결하는 자동화가 어려운 구조적 한계로 인하여
데이터, 문서, 시스템, API가 분산되어 있어 업무 흐름을 하나의 구조로 통합하기 어렵고
자동화 구현 시 개발 의존도가 높은 문제가 존재함.

또한 현업 사용자가 직접 자동화를 구성할 수 있는 도구가 부족하여 단순 반복 업무도 개발 요청을 통해 처리해야 하는 비효율 발생함.

AI 기능이 도입되더라도 단일 기능 중심으로 제공되는 경우가 많아
업무 흐름 전체를 자동화하거나 여러 시스템을 연결하는 데 한계 존재함.

특히 조직 내에서는 데이터는 데이터 시스템에 존재하고 정책은 문서로 관리되며
업무 액션은 별도 시스템에서 수행되고 자동화 로직은 개발 코드로 관리되는 분리된 구조로 인해
업무 프로세스를 유연하게 변경하기 어려운 문제가 있음.

또한 자동화 로직이 코드 중심으로 관리될 경우 현업 사용자가 직접 수정하거나 확장하기 어려워
자동화 활용도가 낮아지는 문제가 발생함.

따라서 데이터, 정책, 액션, AI 기능을 하나의 구조로 연결하고
사용자가 직접 에이전트를 구성하여 업무 자동화를 수행할 수 있는 플랫폼 필요함.

4.3 해결 전략 설계

데이터, 정책, 액션을 조합하여 에이전트를 생성할 수 있는 구조로 플랫폼을 설계함.
소스 등록, 정책 정의, 액션 설정을 통해 업무 자동화 로직을 구성할 수 있도록 에이전트 조립형 구조로 설계함.
현업 사용자가 직접 자동화를 구성할 수 있도록 노코드 / 로우코드 기반 인터페이스로 구성하고 개발 의존도를 낮출 수 있도록 설계함.
AI 기반 빌더를 통해 간단한 설정만으로 프로그램을 자동 생성할 수 있도록 하고 생성된 프로그램을 커스터마이징할 수 있는 구조를 설계함.
에이전트를 워크스페이스에서 실행하고 템플릿 형태로 재사용할 수 있도록 구성함.

또한 대시보드를 통해 에이전트 상태, 주요 지표, 실행 결과를 한눈에 확인할 수 있도록 설계함.
에이전트를 협업 환경에 직접 참여시켜 회의, 채팅, 업무 흐름에서 AI가 실시간으로 개입할 수 있는 구조를 구성함.
이를 통해 업무 자동화 협업 지원 프로그램 생성 데이터 활용을 하나의 플랫폼에서 수행할 수 있도록 설계함.

4.4 실행

AI Orchestration Platform 구조의 적용 가능성을 검증하기 위해
대시보드, 스튜디오, 워크스페이스, AI 빌더 기능을 포함한
플랫폼 프로토타입을 구현함.

스튜디오에서 소스 등록, 정책 설정, 액션 정의를 수행하고 해당 요소를 조합하여 에이전트를 생성할 수 있도록 구성함.
워크스페이스에서 템플릿 및 에이전트를 실행하고 사용자가 직접 자동화 흐름을 구성할 수 있도록 구현함.
